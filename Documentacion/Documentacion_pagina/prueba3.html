SRS – Especificación de Requerimientos

Documento que describe los requerimientos funcionales y no funcionales de la Plataforma de Gestión Hotelera Offline con Dashboard Móvil para el Hotel Santa Ana.

Versión: 1.0
Proyecto: HotelSoft
Equipo: Grupo 2 – SENA

1. Propósito

El propósito de este documento es definir de manera clara y detallada los requerimientos del sistema hotelero, el cual busca optimizar las operaciones diarias del hotel mediante una aplicación de escritorio offline, complementada con un dashboard móvil para supervisión administrativa.

2. Alcance

HotelSoft está orientado al Hotel Santa Ana, el cual requiere un sistema confiable que funcione sin conexión a internet en el área operativa. El sistema está compuesto por dos módulos principales:

  • Aplicación de escritorio offline: destinada al personal operativo para la gestión de huéspedes, habitaciones, reservas, check-in, check-out, pagos y facturación.
  • Dashboard móvil: orientado al dueño y administradores para la visualización de métricas, reportes y alertas administrativas.

3. Descripción General del Sistema

HotelSoft es un sistema integral compuesto por una aplicación de escritorio conectada a una base de datos local y un dashboard móvil que actúa como módulo complementario para la supervisión del negocio.

4. Actores del Sistema

  • Administrador: acceso total a gestión de usuarios, backups, configuración y reportes.
  • Recepcionista: gestión de reservas, check-in, check-out y consultas.
  • Dueño / Administrador General: acceso a métricas, reportes y alertas mediante el dashboard móvil.

5. Requerimientos Funcionales (Resumen)

  • RF-001: Gestión de usuarios (CRUD – solo administrador).
  • RF-003: Gestión de habitaciones y estados.
  • RF-004: Registro de check-in.
  • RF-015: Registro de check-out con generación de factura.
  • RF-016: Gestión de reservas manuales.
  • RF-017: Registro de pagos.
  • RF-018: Visualización de métricas en dashboard móvil.
  • RF-019: Generación de reportes administrativos.
  • RF-020: Exportación y sincronización de datos al dashboard móvil.

6. Requerimientos No Funcionales (Resumen)

  • RNF-001: Funcionamiento offline del sistema.
  • RNF-002: Tiempo de respuesta menor a 2 segundos.
  • RNF-003: Interfaz intuitiva para usuarios no técnicos.
  • RNF-004: Cifrado de copias de seguridad (AES-256).
  • RNF-006: Compatibilidad con sistema operativo Windows.
  • RNF-007: Código modular y mantenible.
  • RNF-008: Auditoría mediante registros de logs.

7. Requerimientos de Interfaz de Usuario

  • UI-001: Login con roles (administrador / recepcionista).
  • UI-003: Panel de recepcionista con check-in y check-out.
  • UI-004: Panel de administrador para gestión y backups.
  • UI-005: Dashboard móvil con métricas y reportes.
